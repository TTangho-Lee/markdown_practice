# 개요
## 1. 철학
- 목적: 읽기 쉽고 쓰기 쉬운 텍스트 형식 지향
- 핵심 가치: 가독성 — 렌더링 없이 평문 상태에서도 서식이 직관적으로 드러나야 함
- 기원: 일반 텍스트 이메일 형식에서 영감을 얻음
- 참조 모델: Setext, atx, Textile, reStructuredText 등 기존 텍스트-HTML 필터
- 구성: 의미를 시각적으로 유추할 수 있는 직관적인 구두점 활용
- 문법 예시
  |목적|방식|
  |-|-|
  |강조|별표를 통해 단어 부각|
  |리스트|실제 목록 형태와 동일|
  |인용구|이메일의 인용 표기 방식 계승|
------------------------------------------------------------------------------------------------------------------------------------------------
## 2. 인라인 HTML
- 설계 목적: 웹용 글쓰기 형식
  - 핵심 정의: 마크다운은 HTML의 대체재가 아닌, 웹을 위한 글쓰기 도구
  - 철학적 차이
    - HTML: 최종 결과물을 보여주는 출판 형식
    - 마크다운: 초안을 작성하고 수정하기 쉬운 집필 형식
- HTML과의 관계
  - 마크다운은 HTML의 매우 작은 부분집합만 다룸
  - 마크다운 구문으로 표현하기 힘든 복잡한 서식은 직접 HTML 태그를 사용하는 것을 권장
- HTML 태그 혼용 시 준수 사항
  |구분|대상 태그 예시| 규칙| 마크다운 구무ㄴ 처리|
  |-|-|-|-|
  |블록 수준|`div`, `table`|주변 텍스트와 빈 줄로 구분, 들여쓰기 금지|태그 내부 마크다운 작동 안 함|
  |스팬 수준|`span`, `img`|단락, 목록, 헤더 내 어디서든 자유롭게 사용 가능|태그 내부 마크다운 작동함|
- 주요 제한 사항 및 주의점
  - 블록 태그 독립성: 마크다운 프로세서는 블록 수준 HTML 태그를 건드리지 않으므로, 태그 내부의 텍스트에 마크다운 문법을 적용할 수 없음
  - 작성 편의성: 링크나 이미지 등 마크다운 문법이 기억나지 않을 경우, 익숙한 HTML 태그를 그대로 사용해도 무방함
------------------------------------------------------------------------------------------------------------------------------------------------
## 3. 특수 문자에 대한 자동 이스케이핑
- 핵심 문제: HTML에서 `<`와 `&`는 태그와 엔티티를 나타내는 예약어로, 문자 그대로 쓰기 위해서는 번거로운 치환 과정이 필수적임
- 해결 방안: 마크다운은 사용자가 특수 문자를 자연스럽게 사용하더라도 이를 상황에 맞게 자동 인코딩하여 작성자의 실수를 방지하고 생산성을 높임
- 주요 문자 처리 메커니즘
  |역할|문자|HTML에서의 문제|마크다운의 처리방식|
  |-|-|-|-|
  |엔티|`&`|`AT&T` → `AT&amp;T`|일반 텍스트로 쓰면 자동 변환|
  |태그|`<`|`4 < 5` → `4 &lt; 5`|HTML 태그 구분자가 아닐 경우 자동으로 변환|
- 마크다운의 지능적 변환 규칙
  - 유연한 수용: 사용자가 직접 HTML 엔티티를 입력하면 마크다운은 이를 변경하지 않고 그대로 둠
  - 자동 변환: 일반적인 문맥에서 `&`나 `<`를 사용하면 마크다운이 이를 인식하여 안전한 HTML 코드로 자동 번역함
  - URL 관리: 복잡한 URL 내에 포함된 `&` 기호를 일일이 변환할 필요 없이 평문 그대로 작성 가능
- 코드 작성에서의 강점
  - 자동 인코딩: 마크다운의 코드 구문 내부에서는 `<`와 `&`가 항상 자동으로 인코딩됨
  - 비교 우위: HTML로 HTML 예제 코드를 작성하는 것은 모든 태그를 이스케이프해야 해서 힘들지만, 마크다운을 사용하면 원본 코드를 그대로 복사해서 붙여넣는 것만으로도 완벽하게 표시됨
------------------------------------------------------------------------------------------------------------------------------------------------
# 블록 요소
## 1. 단락과 줄 바꿈
- 단락의 정의 및 구성
  - 형성조건: 하나 이상의 연속된 텍스트 행이 모여 하나의 단락을 이룸
  - 구분 방법: 단락과 단락 사이는 하나 이상의 빈 줄로 구분
    - 완전히 비어 있거나, 공백 및 탭만 포함된 줄도 빈 줄로 간주함
  - 주의 사항: 일반적인 단락은 첫 줄을 공백이나 탭으로 들여쓰기 하지 않음
- 하드랩 텍스트 지원
  - 마크다운은 편집기 내에서 여러 줄로 나누어 작성된 텍스트를 하나의 단락으로 인식함
  - 이는 이메일 스타일의 블록 인용구나 복잡한 목록 항목을 작성할 때 시각적인 가독성을 높여주는 핵심 장치
- 줄 바꿈 처리 규칙
  - 마크다운은 다른 텍스트 포맷터처럼 모든 엔터를 `<br />` 태그로 변환하지 않음
    |구분|방법|결과|
    |-|-|-|
    |일반|엔터줄 끝에서 바로 `Enter`|렌더링 시 공백으로 처리|
    강제|줄 바꿈줄 끝에 두 칸 이상의 공백 입력 후 엔터|`<br />` 태그 삽입|
- 설계 의도 및 차별점
  - 가독성 유지: 텍스트 내에서 무분별한 `<br />` 태그 발생을 막아 문서를 깔끔하게 유지함
  - 이메일 형식 계승: 과거 텍스트 기반 이메일에서 사용하던 관습을 그대로 가져와, 작성자가 의도한 위치에서만 정교하게 줄을 바꿀 수 있도록 설계됨
------------------------------------------------------------------------------------------------------------------------------------------------
## 2. 헤더
- Setext 스타일 (밑줄 방식)
  - 주로 문서의 큰 제목과 중간 제목을 표현할 때 사용하며, 텍스트 아래에 기호를 받쳐주는 형태
  - 1단계 헤더 (H1): 등호(`=`)를 사용하여 밑줄을 그음
  - 2단계 헤더 (H2): 대시(`-`)를 사용하여 밑줄을 그음
  - 특징: 밑줄 기호의 개수는 텍스트 길이와 꼭 맞출 필요 없이 하나 이상만 사용하면 작동
- Atx 스타일 (해시 방식)
  - 가장 널리 쓰이는 방식으로, 줄 시작 부분에 샵(`#`) 기호를 붙여 1단계부터 6단계까지의 헤더 레벨을 결정
    |헤더 레벨|사용 기호|예시|
    |-|-|-|
    |H1|`#`|`# 제목 1`|
    |H2|`##`|`## 제목 2`|
    |...|...|...|
    |H6|`######`|`###### 제목 6`|
- Atx 스타일의 장식용 닫기 기능
  - Atx 스타일은 시각적인 대칭을 위해 헤더 끝에 닫는 해시(`#`)를 추가할 수 있음
  - 심미적 용도: 닫는 해시는 순전히 가독성과 디자인을 위한 것이며, 헤더 레벨에 영향을 주지 않음
  - 유연성: 시작하는 해시의 개수가 헤더 레벨을 결정하며, 끝나는 해시의 개수는 시작점과 달라도 무방함
------------------------------------------------------------------------------------------------------------------------------------------------
## 3. 블록인용
- 기원: 전통적인 텍스트 기반 이메일의 인용 방식을 계승함
- 기본 문법: 인용하려는 문장이나 단락 앞에 > 기호를 삽입함
- 작성 방식의 유연성
  - 마크다운은 작성자의 편의를 위해 두 가지 스타일의 작성을 모두 지원함
    |구분|작성 방법|특징|
    |-|-|-|
    |Hard|모든 줄 앞에 `>` 삽입|시각적으로 깔끔하며 인용구임을 명확히 인지 가능|
    |Lazy|단락의 첫 줄에만 `>` 삽입|연속된 텍스트의 경우 첫 줄만 표시해도 전체가 인용 처리됨|
- 중첩 및 복합 요소 포함
  - 블록 인용은 단순히 텍스트를 담는 것을 넘어, 다른 마크다운 요소들과 결합하여 풍부한 표현이 가능
  - 중첩 인용: `> >`와 같이 기호를 중첩하여 인용구 안에 또 다른 인용구를 넣을 수 있음
  - 요소 포함: 인용구 내부에서 헤더, 리스트, 코드 블록 등 거의 모든 마크다운 문법을 그대로 사용할 수 있음
- 편집기 활용
  - 많은 텍스트 편집기에서 인용 레벨 증가 기능을 통해 선택한 영역을 한 번에 블록 인용구로 변환할 수 있어 생산성이 높음
------------------------------------------------------------------------------------------------------------------------------------------------
## 4. 목록
- 무순서 목록
  - 사용 기호: `*`, `+`, `-`을 혼용하거나 교환하여 사용 가능
  - 특징: 기호 종류에 상관없이 동일한 불릿 포인트 형태로 렌더링
- 순서 목록
  - 사용 방법: 숫자 뒤에 마침표를 붙여 작성
  - 자동 번호 매김: 소스 코드에 어떤 숫자를 쓰더라도 HTML 출력 시에는 자동으로 순차적인 번호가 매겨짐
  - 주의 사항: 목록은 항상 숫자 1로 시작하는 것이 권장
- 목록의 레이아웃 및 들여쓰기
  - 시작 여백: 목록 기호는 줄 시작점에서 최대 3칸까지 들여쓸 수 있으며, 기호 뒤에는 반드시 하나 이상의 공백이나 탭이 필요
  - 가독성: 긴 문장을 작성할 때 다음 줄을 목록 기호 위치에 맞춰 들여쓰면 보기 좋지만, 들여쓰지 않아도 마크다운은 이를 하나의 항목으로 인식
- 복합 요소 포함 및 간격 규칙
  |구분|처리 규칙|
  |-|-|
  |단락 구분|항목 사이가 빈 줄로 구분되면 각 항목은 HTML의 `<p>` 태그로 감싸짐|
  |다중 단락|목록 내에서 단락을 나누려면, 다음 단락을 4칸의 공백이나 1개의 탭으로 들여써야 함|
  |블록 인용구|인용 기호 앞에 들여쓰기를 적용하여 목록 내에 포함|
  |코드 블록|목록 내에 코드 블록을 넣으려면 8칸의 공백이나 2개의 탭을 들여써야 함|
- 예외 상황 처리: 의도치 않은 목록 생성 방지
  - 문제: "1986. "과 같이 숫자로 시작하는 문장을 쓰면 마크다운이 이를 순서 목록으로 오인할 수 있음.
  - 해결: 마침표 앞에 `\`를 사용하여 목록 생성을 방지
------------------------------------------------------------------------------------------------------------------------------------------------
## 5. 코드 블록
- 개요
  - Markdown에서 코드 블록은 프로그래밍 소스 코드나 마크업 언어를 본문과 분리하여 있는 그대로 보여주기 위해 사용됨
  - 일반 단락과 달리 정해진 형식을 유지하며 HTML의 `<pre><code>` 태그로 변환
- 작성 방법: 들여쓰기 활용
  - 가장 기본적인 코드 블록 생성 방법은 들여쓰기를 이용하는 것
  - 방법: 각 줄의 시작 부분에 최소 4개의 공백 또는 1개의 탭을 삽입
  - 종료: 들여쓰기가 없는 줄이 나타나면 코드 블록이 종료된 것으로 간주
  - 결과: 실제 출력 시에는 코드 블록을 만들기 위해 사용된 첫 단계의 들여쓰기는 제거되고 나머지 내용만 표시
- 주요 특징 및 장점
  - 자동 HTML 엔티티 변환
    - 코드 블록 내에서는 특수 기호들이 자동으로 HTML 엔티티로 변환되어 브라우저에서 안전하게 표시
    - `&`: `&amp;`로 변환
    - `<`: `&lt;`로 변환
    - `>`: `&gt;`로 변환
    - HTML 소스 코드를 별도의 인코딩 작업 없이 그대로 복사해서 붙여넣기만 하면 되므로 매우 편리
  - Markdown 문법 무시
    - 코드 블록 내부에서는 Markdown의 일반적인 서식 문법이 적용되지 않음
    - Markdown 자체의 문법을 설명하는 글을 작성하기에도 용이
------------------------------------------------------------------------------------------------------------------------------------------------
## 6. 수평 규칙
- 문서 내에서 시각적인 구분선을 생성하여 내용의 단락이나 주제가 바뀌었음을 알리는 역할
- HTML로 변환될 때 `<hr />` 태그로 렌더링
  - 사용 가능 기호: `*`, `-`, `_`
  - 최소 수량: 한 줄에 해당 기호를 3개 이상 연속으로 배치
  - 공백 허용: 기호와 기호 사이에 공백을 삽입해도 동일하게 수평 규칙으로 인식
  - 주의 사항: 해당 줄에는 다른 텍스트 없이 오직 기호와 공백만 존재해야 함

# 스팬 요소
## 1. 링크
- Markdown은 웹 페이지나 로컬 리소스로 연결되는 링크를 만들기 위해 두 가지 스타일을 제공
  - 인라인 링크: 본문 내에 즉시 URL을 기입하는 방식
  - 참조 스타일 링크: 본문에는 식별자만 남기고, 실제 URL 정보는 문서 다른 곳에 정의하는 방식
- 인라인 링크
  - 구문: `[링크 텍스트](URL "제목")`
  - 특징: 제목은 따옴표로 감싸며 생략 가능
  - 서버 내 리소스를 연결할 때는 상대 경로를 사용ㄱ능
- 참조 스타일 링크
  - 문서의 가독성을 극대화하기 위해 URL 본문 밖으로 분리하는 방식
  - 본문 내 작성
    - `[링크 텍스트][ID]` 형식
    - 괄호 사이에 공백이 있어도 무방
  - 링크 정의
    - 문서 내 어디에서든 URL을 정의
    - 형식: `[ID]: URL "제목"`
    - 규칙
      - ID는 대소문자를 구분하지 않음
      - URL은 `< >`으로 감쌀 수 있음
      - 제목은 `""`, `''`, `(())`로 감쌀 수 있음
  - 암묵적 링크 이름
    - 링크 텍스트 자체가 ID가 될 경우 빈 대괄호를 사용하여 단축 가능
    - 본문: `[Google][]`
    - 정의: `[Google]: http://google.com`
------------------------------------------------------------------------------------------------------------------------------------------------
## 2. 강조점
- 텍스트에 시각적인 무게감을 더하기 위해 사용
- 강조하고자 하는 텍스트를 특정 기호로 감싸서 표현하며, 시작과 끝은 반드시 동일한 기호를 사용
  |효과|작성 방법|
  |-|-|
  |이탤릭 (Italic)|*텍스트* 또는 _텍스트_|
  |볼드 (Bold)|**텍스트** 또는 __텍스트__|
- 세부 규칙 및 주의 사항
  - 단어 중간 강조: 강조 문법은 단어 전체뿐만 아니라 단어의 일부분에도 적용 가능
  - 공백에 따른 리터럴 처리: 강조 기호의 양옆에 공백이 있는 경우, Markdown은 이를 강조 기호가 아닌 문자 그대로의 별표나 밑줄로 인식
  - 백슬래시 이스케이프: 강조 기호로 쓰이는 문자를 화면에 그대로 출력하고 싶을 때는 기호 앞에 `\`를 붙여서 기능을 무효화

## 3. 코드
- 코드 스팬은 일반적인 문장 중간에 짧은 코드나 명령어, 변수명 등을 삽입할 때 사용
- 기본 작성법: 텍스트를 백틱(`` ` ``)기호로 감싸서 표현
- 코드 내에 백틱 포함하기
  - 코드 스팬 안에 문자 그대로의 백틱(`)을 표시해야 할 경우, 시작과 끝 구분자를 두 개 이상의 백틱으로 사용
    - 방법: 더 많은 수의 백틱으로 감싼다
    - 예시: ``백틱(`)`` 
  - 공백을 이용한 구분
    - 코드 스팬의 시작이나 끝에 백틱 문자가 위치해야 한다면, 구분자와 내용 사이에 한 칸의 공백을 삽입하여 인식 오류를 방지한다
    - 예시 1:``백틱` ``
    - 예시 2:`` `백틱` ``
- 주요 특징: 자동 엔티티 인코딩코드 스팬 내에서는 특수 기호들이 자동으로 HTML 엔티티로 변환됨

## 4. 이미지
- 인라인 스타일 이미지
  - 이미지 정보를 본문에 즉시 기입하는 가장 보편적인 방식
  - 구문: `![텍스트](URL "제목")`
  - 구성 요소
    - 느낌표(!): 이미지임을 나타내는 시작 기호
    - 대괄호([ ]): 이미지의 alt 속성을 입력
    - 괄호(( )): 이미지의 경로(URL)와 따옴표로 감싼 제목(생략가능)을 입력
- 참조 스타일 이미지
  - 가독성을 위해 본문에는 식별자만 남기고, 이미지의 상세 경로는 문서 하단 등에 별도로 정의하는 방식
  - 본문 내 작성: `![텍스트][id]`
  - 이미지 정의: `[id]: URL "제목"`
  - 링크 정의 방식과 동일하게 작동하며, 식별자(id)를 통해 본문의 마크업과 실제 경로를 연결
- 한계점 및 대안
  - 표준 Markdown 문법에는 이미지의 크기를 직접 조절하는 구문이 포함되어 있지 않음
  - 해결책: 이미지 크기 조절이 꼭 필요한 경우, Markdown 대신 일반 HTML 태그인 `<img src="..." width="..." height="..." />`를 사용

## 1. 백슬래시 탈출
- 기호를 서식 용도가 아닌 문자 그대로 화면에 표시해야 할 때, 백슬래시(`\`)를 앞에 붙여 문법 기능을 무효화
- 작동 원리: 강조나 목록 생성에 사용되는 기호 바로 앞에 백슬래시(`\`)를 삽입하면, Markdown 엔진은 해당 기호를 문법으로 해석하지 않고 일반 텍스트로 처리
- Markdown은 다음 12가지 기호에 대해 백슬래시 이스케이프를 제공
  |기호|명칭|기호|명칭|
  |-|-|-|-|
  |\\|백슬래시|\`|백틱|
  |\*|별표|\_|밑줄|
  |\{ \}|중괄호|\[ \]|대괄호|
  |\( \)|소괄호|\#|해시 마크|
  |\+|더하기 기호|\-|빼기/하이픈|
  |\.|마침표|\!|느낌표|
