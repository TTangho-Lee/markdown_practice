# 개요
## 1. 철학
- 목적: 읽기 쉽고 쓰기 쉬운 텍스트 형식 지향
- 핵심 가치: 가독성 — 렌더링 없이 평문 상태에서도 서식이 직관적으로 드러나야 함
- 기원: 일반 텍스트 이메일 형식에서 영감을 얻음
- 참조 모델: Setext, atx, Textile, reStructuredText 등 기존 텍스트-HTML 필터
- 구성: 의미를 시각적으로 유추할 수 있는 직관적인 구두점 활용
- 문법 예시
  |목적|방식|
  |-|-|
  |강조|별표를 통해 단어 부각|
  |리스트|실제 목록 형태와 동일|
  |인용구|이메일의 인용 표기 방식 계승|

## 2. 인라인 HTML
- 설계 목적: 웹용 글쓰기 형식
  - 핵심 정의: 마크다운은 HTML의 대체재가 아닌, 웹을 위한 글쓰기 도구
  - 철학적 차이
    - HTML: 최종 결과물을 보여주는 출판 형식
    - 마크다운: 초안을 작성하고 수정하기 쉬운 집필 형식
- HTML과의 관계
  - 마크다운은 HTML의 매우 작은 부분집합만 다룸
  - 마크다운 구문으로 표현하기 힘든 복잡한 서식은 직접 HTML 태그를 사용하는 것을 권장
- HTML 태그 혼용 시 준수 사항
  |구분|대상 태그 예시| 규칙| 마크다운 구무ㄴ 처리|
  |-|-|-|-|
  |블록 수준|`div`, `table`|주변 텍스트와 빈 줄로 구분, 들여쓰기 금지|태그 내부 마크다운 작동 안 함|
  |스팬 수준|`span`, `img`|단락, 목록, 헤더 내 어디서든 자유롭게 사용 가능|태그 내부 마크다운 작동함|
- 주요 제한 사항 및 주의점
  - 블록 태그 독립성: 마크다운 프로세서는 블록 수준 HTML 태그를 건드리지 않으므로, 태그 내부의 텍스트에 마크다운 문법을 적용할 수 없음
  - 작성 편의성: 링크나 이미지 등 마크다운 문법이 기억나지 않을 경우, 익숙한 HTML 태그를 그대로 사용해도 무방함

## 3. 특수 문자에 대한 자동 이스케이핑
- 핵심 문제: HTML에서 `<`와 `&`는 태그와 엔티티를 나타내는 예약어로, 문자 그대로 쓰기 위해서는 번거로운 치환 과정이 필수적임
- 해결 방안: 마크다운은 사용자가 특수 문자를 자연스럽게 사용하더라도 이를 상황에 맞게 자동 인코딩하여 작성자의 실수를 방지하고 생산성을 높임
- 주요 문자 처리 메커니즘
  |역할|문자|HTML에서의 문제|마크다운의 처리방식|
  |-|-|-|-|
  |엔티|&|AT&T → AT&amp;T|일반 텍스트로 쓰면 자동 변환|
  |태그|<|4 < 5 → 4 &lt; 5|HTML 태그 구분자가 아닐 경우 자동으로 변환|
- 마크다운의 지능적 변환 규칙
  - 유연한 수용: 사용자가 직접 HTML 엔티티를 입력하면 마크다운은 이를 변경하지 않고 그대로 둠
  - 자동 변환: 일반적인 문맥에서 &나 <를 사용하면 마크다운이 이를 인식하여 안전한 HTML 코드로 자동 번역함
  - URL 관리: 복잡한 URL 내에 포함된 & 기호를 일일이 변환할 필요 없이 평문 그대로 작성 가능
- 코드 작성에서의 강점
  - 자동 인코딩: 마크다운의 코드 구문 내부에서는 <와 &가 항상 자동으로 인코딩됨
  - 비교 우위: HTML로 HTML 예제 코드를 작성하는 것은 모든 태그를 이스케이프해야 해서 힘들지만, 마크다운을 사용하면 원본 코드를 그대로 복사해서 붙여넣는 것만으로도 완벽하게 표시됨

# 블록 요소
## 1. 단락과 줄 바꿈
- 단락의 정의 및 구성
  - 형성조건: 하나 이상의 연속된 텍스트 행이 모여 하나의 단락을 이룸
  - 구분 방법: 단락과 단락 사이는 하나 이상의 빈 줄로 구분
    - 완전히 비어 있거나, 공백 및 탭만 포함된 줄도 빈 줄로 간주함
  - 주의 사항: 일반적인 단락은 첫 줄을 공백이나 탭으로 들여쓰기 하지 않음
- 하드랩 텍스트 지원
  - 마크다운은 편집기 내에서 여러 줄로 나누어 작성된 텍스트를 하나의 단락으로 인식함
  - 이는 이메일 스타일의 블록 인용구나 복잡한 목록 항목을 작성할 때 시각적인 가독성을 높여주는 핵심 장치
- 줄 바꿈 처리 규칙
  - 마크다운은 다른 텍스트 포맷터처럼 모든 엔터를 `<br />` 태그로 변환하지 않음
    |구분|방법|결과|
    |-|-|-|
    |일반|엔터줄 끝에서 바로 Enter|렌더링 시 공백으로 처리|
    강제|줄 바꿈줄 끝에 두 칸 이상의 공백 입력 후 엔터|`<br />` 태그 삽입|
- 설계 의도 및 차별점
  - 가독성 유지: 텍스트 내에서 무분별한 `<br />` 태그 발생을 막아 문서를 깔끔하게 유지함
  - 이메일 형식 계승: 과거 텍스트 기반 이메일에서 사용하던 관습을 그대로 가져와, 작성자가 의도한 위치에서만 정교하게 줄을 바꿀 수 있도록 설계됨

## 2. 헤더
- Setext 스타일 (밑줄 방식)
  - 주로 문서의 큰 제목과 중간 제목을 표현할 때 사용하며, 텍스트 아래에 기호를 받쳐주는 형태
  - 1단계 헤더 (H1): 등호(=)를 사용하여 밑줄을 그음
  - 2단계 헤더 (H2): 대시(-)를 사용하여 밑줄을 그음
  - 특징: 밑줄 기호의 개수는 텍스트 길이와 꼭 맞출 필요 없이 하나 이상만 사용하면 작동
- Atx 스타일 (해시 방식)
  - 가장 널리 쓰이는 방식으로, 줄 시작 부분에 샵(#) 기호를 붙여 1단계부터 6단계까지의 헤더 레벨을 결정
    |헤더 레벨|사용 기호|예시|
    |-|-|-|
    |H1|#|# 제목 1|
    |H2|##|## 제목 2|
    |.
    .|.
    .|.
    .|
    |H6|######|###### 제목 6|
- Atx 스타일의 장식용 닫기 기능
  - Atx 스타일은 시각적인 대칭을 위해 헤더 끝에 닫는 해시(#)를 추가할 수 있음
  - 심미적 용도: 닫는 해시는 순전히 가독성과 디자인을 위한 것이며, 헤더 레벨에 영향을 주지 않음
  - 유연성: 시작하는 해시의 개수가 헤더 레벨을 결정하며, 끝나는 해시의 개수는 시작점과 달라도 무방함
